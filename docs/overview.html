PostWRF as a Linux-based software, is comprised of two main components: NCAR Command Language (NCL; http://dx.doi.org/10.5065/D6WD3XH5), and Linux Bourne Again Shell (Bash; https://www.gnu.org/software/bash/) scripts. All the post-processing tasks, such as drawing contour maps and extracting the time series of data over single points, are carried out by NCL. For each post-processing task and visualization, there are one or more corresponding NCL scripts which should be modified appropriately according to that specific task. By the virtue of the Linux shell script, the end users do not need to make any direct modifications on the NCL scripts. In other words, Linux shell scripts act as proxies between NCL codes and the end user, to modify the NCL codes with regard to some primary input data, provided by the user. It should be noted that PostWRF is not a fully automated system, because the user has some primary control over how the software is run and the processes are carried out.
